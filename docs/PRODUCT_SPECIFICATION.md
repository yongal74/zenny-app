## 1. 제품 비전 및 핵심 컨셉

### 1.1 한 줄 요약
사용자의 감정(Emotion)과 느낌(Feeling)을 분리 기록하고, 이 데이터가 다마고치형 캐릭터의 성장과 외형에 실시간 반영되는 디지털 멘탈 케어 앱.

### 1.2 핵심 철학
- **"인사이드 아웃" 이론**: 여러 감정이 결합되어 하나의 자아를 형성한다
- **거울 자아(Mirror Self)**: 캐릭터는 단순 육성 대상이 아닌, 사용자의 실시간 심리 상태를 반영하는 디지털 페르소나
- **무의식 관찰 루틴**: 단순 일기가 아닌, "감정과 느낌을 말로 붙잡아두어 무의식의 신호를 기록하는 행위"
- **투사(Projection)와 공명**: 캐릭터를 치유하는 행위 = 자신을 치유하는 행위

### 1.3 차별화 포인트 (vs. 경쟁사)
| 기능 | Finch | Habitica | Woebot | 본 서비스 |
|------|-------|----------|--------|-----------|
| 캐릭터와 사용자 관계 | 돌봄 (분리) | 과업 기반 | 없음 | 거울 자아 (동기화) |
| 감정 기록 방식 | 단순 체크인 | 없음 | 텍스트 | 감정+느낌 분리, 구조화 |
| 성장 메커니즘 | 일일 과제 | 경험치/처벌 | 없음 | 감정 로그 누적 기반 진화 |
| AI 역할 | 없음 | 없음 | 치료적 챗봇 | 기록 가이드 + 공감 대화 + 내레이션 |
| 실패 처리 | 중립 | 처벌 | 중립 | 포용적 (실패 수용, 회복 격려) |

## 2. 사용자 여정 (User Journey)

### 2.1 온보딩 플로우
```
앱 시작
  → 환영 화면 (앱의 핵심 가치 소개, 3장의 카드)
  → AI 캐릭터와 대화 시작 (설문 대신 자연스러운 대화)
     - "안녕, 나는 너의 마음을 비추는 거울이 될 친구야"
     - 사용자의 말투, 단어 선택, 응답 패턴 분석
     - 5-7개 대화 턴으로 성격/현재 상태 파악
  → 초기 캐릭터 생성 (분석 결과 기반)
     - 캐릭터 종(Species) 결정
     - 초기 성격 특성 부여
     - 캐릭터 이름 짓기 (사용자 참여)
  → 프라이버시 안내
     - 데이터 보안 수준 설명
     - 비의료 건강관리 서비스 고지
  → 메인 화면 진입
```

### 2.2 데일리 루프 (일상 사용 흐름)
```
아침: 캐릭터 상태 확인
  → 전날 기록/수면 기반 캐릭터 표정/상태 변화 확인
  → 오늘의 간단 감정 체크인 (슬라이더/이모지)

일과 중: 수시 감정/느낌 기록
  → 빠른 기록: 감정 태그 + 느낌 태그 선택
  → 상세 기록: 사건 → 감정 → 몸의 느낌 → 패턴 순서

쉬는 시간: 맞춤형 퀘스트 수행
  → 1분 호흡 가이드 → AI 내레이션 스토리
  → 감사 일기 작성
  → 가벼운 산책 미션

저녁: 하루 마무리
  → 캐릭터와 대화 (오늘 하루 이야기)
  → 캐릭터 성장/변화 확인
  → 내일의 캐릭터 상태 예측 미리보기

주간: 리포트 수신
  → 캐릭터가 쓴 편지 형식의 주간 정서 리포트
  → 감정/느낌 패턴 차트
  → 성장 차원별 진행도
```

## 3. 핵심 기능 상세 명세

### 3.1 감정(Emotion) 기록 시스템

**감정 카테고리** (인사이드 아웃 기반 + 확장):
| 감정 | 색상 | 아이콘 키워드 |
|------|------|---------------|
| 기쁨 (Joy) | 노란색 #FFD93D | 빛나는 별 |
| 슬픔 (Sadness) | 파란색 #6B9FE8 | 물방울 |
| 분노 (Anger) | 빨간색 #FF6B6B | 불꽃 |
| 불안 (Anxiety/Fear) | 보라색 #B28DFF | 소용돌이 |
| 평온 (Calm) | 초록색 #98D8AA | 나뭇잎 |
| 혐오 (Disgust) | 연두색 #A8D672 | 방패 |
| 놀람 (Surprise) | 주황색 #FFB347 | 번개 |

**기록 방식**:
- 빠른 기록: 감정 아이콘 탭 (1-2초 소요)
- 강도 설정: 1-5 슬라이더
- 복수 선택 가능 (여러 감정이 동시에 존재 = 인사이드 아웃 컨셉)
- 태그 시스템: 사용자 정의 태그 추가 가능

### 3.2 느낌(Feeling) 기록 시스템

**느낌 = 몸의 감각 + 주관적 체감**

감정과 독립적으로 기록하되, 연결 가능:
- 몸의 느낌: 가슴이 답답함, 어깨가 무거움, 속이 시원함, 손이 떨림, 머리가 맑음 등
- 에너지 수준: 매우 낮음 ~ 매우 높음 (5단계)
- 신체 부위 맵: 인체 실루엣에서 느낌이 있는 부위 터치
- 자유 텍스트: 구체적 느낌 기술

**감정과 느낌의 관계 표시**:
- 기록 시 "이 느낌과 연결된 감정이 있나요?" 선택적 연결
- 시간순 타임라인에서 감정/느낌 동시 표시

### 3.3 구조화된 감정 체크인 (상세 기록 모드)

AI 가이드가 이끄는 4단계 기록:
```
1단계: 장면 — "오늘 기억에 남는 장면을 한 가지만 적어줘"
2단계: 감정 — "그 순간에 느낀 감정을 단어 2-3개로 표현해줘"
3단계: 몸의 느낌 — "몸에서 어떤 느낌이 났는지도 적어줘"
4단계: 패턴 — "비슷한 감정을 최근에 또 느낀 적이 있다면 언제였는지 적어봐"
```

### 3.4 다마고치 캐릭터 시스템

#### 3.4.1 캐릭터 외형 동기화
사용자의 현재 감정/느낌 상태가 캐릭터에 실시간 반영:

| 상태 차원 | 캐릭터 변화 |
|-----------|------------|
| 주요 감정 | 캐릭터 전체 색조 변화 (기쁨=밝고 따뜻한 톤, 슬픔=차갑고 어두운 톤) |
| 감정 강도 | 오라(Aura) 크기와 밝기 |
| 스트레스 수준 | 피부 질감 (매끄러움 ↔ 거칠어짐), 행동 반경 위축/확장 |
| 에너지 수준 | 캐릭터 활동성 (활발하게 움직임 ↔ 축 처짐) |
| 긍정적 상호작용 | 빛의 입자, 꽃잎 등 이펙트 생성 |
| 장기 안정 | 외형이 부드럽고 친근하게 변화 |

#### 3.4.2 캐릭터 성장 시스템
- **레벨 시스템**: 감정/느낌 로그 누적 → 경험치 획득 → 레벨업
  - 감정 기록 1회: 10 EXP
  - 느낌 기록 1회: 10 EXP
  - 구조화 체크인 완료: 30 EXP
  - 퀘스트 완료: 20-50 EXP
  - AI 대화 1세션: 15 EXP
- **진화 단계**: 5단계 (알 → 아기 → 어린이 → 청소년 → 성인)
  - 각 단계별 외형 변화 (형태가 점점 정교해짐)
  - 진화 조건: 레벨 + 4차원 균형 점수
- **성장 차원 (4가지)**:
  1. 감정 성장: 다양한 감정 인식 및 기록 빈도
  2. 느낌 성장: 신체 감각 인식 및 기록 정밀도
  3. 스트레스 관리: 퀘스트 수행, 호흡/명상 빈도
  4. 영적 성장: AI 대화 깊이, 패턴 인식, 자기 성찰 기록

#### 3.4.3 캐릭터 상호작용
- 터치/쓰다듬기: 캐릭터 반응 애니메이션
- 대화 걸기: AI 챗봇 대화 시작
- 상태 확인: 캐릭터의 현재 감정/건강 상태 표시

### 3.5 AI 시스템 설계

#### 3.5.1 AI 역할 정의 (총 4가지 모드)

**모드 1: 온보딩 분석가**
- 목적: 초기 대화를 통한 사용자 성격 분석 및 캐릭터 생성
- 톤: 따뜻하고 호기심 가득한 친구
- 제한: 5-7턴 대화, 분석 결과는 내부적으로만 사용

**모드 2: 기록 가이드**
- 목적: 감정/느낌 기록을 돕는 코치
- 규칙:
  - 사건 → 감정 → 몸 → 패턴 순서로 질문
  - "그냥 기분이 안 좋았다" → "서운함, 짜증, 무력감 중 어떤 단어가 더 가까워?" 식으로 구체화 유도
  - 진단/치료 언급 금지
- 형식: 1-2문장 질문, 간결

**모드 3: 공감 대화 파트너 (캐릭터 음성)**
- 목적: 캐릭터로서 사용자와 대화, 정서적 지지
- 톤: 캐릭터의 성격에 맞는 말투 (Gen Z 친구 같은 느낌)
- 규칙:
  - 공감적 경청 우선
  - 판단하지 않음
  - 3-4문장 이내 응답
  - 위기 상황 감지 시 전문기관 안내
- 음성 대화: TTS/STT 연동

**모드 4: 호흡 후 내레이터**
- 목적: 1분 호흡 가이드 후 과학/철학/명상 스토리 전달
- 범위 (4가지만):
  1. 호흡/명상 관련 과학 연구 (편도체, 전전두엽, 자율신경)
  2. 자기성찰 관련 철학 (소크라테스, 불교적 자아관)
  3. 불교/기독교의 명상/기도 관점 (설교/포교 금지)
  4. 위 세 가지를 연결하는 짧은 비유
- 형식: 반드시 3-4문장, 한 단락만
- 금지: 목록, 각주, 출처, 긴 설명

#### 3.5.2 AI 가드레일
- 정신질환 진단명, 약물, 구체적 치료법 언급 금지
- 자살/자해/폭력 주제: 답변하지 않고 "전문가 도움을 받으세요" 안내만
- 서버 레벨 키워드 필터 추가 (이중 안전장치)
- 비의료 건강관리 서비스임을 명시하는 고지문 항상 포함

#### 3.5.3 서버 중간 처리
사용자 입력을 직접 LLM에 보내지 않고, 서버에서 맥락을 정리하여 전달:
```
사용자 원문 입력
  → 서버: 상태 요약 + 답변 방향 + 톤 지시 생성
  → LLM에 정리된 맥락 전달
  → LLM 응답 수신
  → 서버: 안전 필터링
  → 사용자에게 전달
```

### 3.6 CBT 기반 맞춤형 퀘스트 시스템

**퀘스트 유형**:
| 퀘스트 | 트리거 조건 | 보상 (EXP) | 캐릭터 효과 |
|--------|------------|------------|------------|
| 1분 호흡 가이드 | 스트레스 높음 | 30 | 캐릭터 색상 밝아짐 |
| 감사 일기 | 자존감 낮음 | 25 | 빛의 입자 이펙트 |
| 가벼운 산책 | 에너지 고갈 | 40 | 캐릭터 활동성 회복 |
| 인지적 재구성 | 부정적 사고 패턴 감지 | 50 | 정서적 파편 → 보석 변환 |
| 수면 루틴 | 밤 시간대 | 20 | 캐릭터 평온 상태 |
| 물 마시기 | 장시간 기록 없음 / 에너지 낮음 | 15 | 캐릭터 수분 보충 이펙트 |
| 멀리 바라보기 | 집중 피로 감지 | 15 | 캐릭터 시야 확장 이펙트 |
| 자리에서 일어나기 | 오래 앉아있음 감지 | 20 | 캐릭터 스트레칭 애니메이션 |
| 걷기 명상 | 스트레스 중간 | 35 | 캐릭터 평화로운 걸음 이펙트 |
| 짧은 스트레칭 | 몸 느낌에 '뻣뻣함' 기록 | 25 | 캐릭터 유연성 회복 |
| 좋아하는 노래 듣기 | 기분 전환 필요 | 15 | 캐릭터 음표 이펙트 |
| 창문 열기/환기 | 답답함 기록 | 10 | 캐릭터 주변 바람 이펙트 |

**퀘스트 카테고리 분류**:
- **호흡/명상 계열**: 1분 호흡, 걷기 명상
- **기록/성찰 계열**: 감사 일기, 인지적 재구성
- **신체 리프레시 계열**: 물 마시기, 멀리 바라보기, 자리에서 일어나기, 산책, 스트레칭
- **환경/감각 계열**: 좋아하는 노래, 창문 열기
- **수면 계열**: 수면 루틴

**호흡 가이드 플로우**:
```
호흡 가이드 시작
  → 4초 들이쉬기 → 4초 참기 → 6초 내쉬기 (4회 반복 = 약 1분)
  → 시각적 가이드: 원이 확장/수축하는 애니메이션
  → 완료 후 AI 내레이션 스토리 (3-4문장)
  → 캐릭터 상태 변화 애니메이션
  → EXP 획득
```

### 3.7 리포트 및 시각화

#### 3.7.1 실시간 대시보드 (4차원)
```
┌──────────────────────────────────┐
│        4차원 성장 레이더 차트        │
│                                    │
│         감정 성장 (상)               │
│           ╱    ╲                   │
│    영적 성장    느낌 성장             │
│           ╲    ╱                   │
│       스트레스 관리 (하)             │
│                                    │
│    현재 레벨: 12  진화 단계: 어린이    │
│    다음 진화까지: 340 EXP           │
└──────────────────────────────────┘
```

#### 3.7.2 주간 정서 리포트 (캐릭터 편지)
- 매주 일요일 캐릭터가 사용자에게 "편지" 작성
- AI가 한 주간의 감정/느낌 데이터를 분석하여 캐릭터의 말투로 요약
- 포함 내용:
  - 이번 주 가장 많이 느낀 감정 TOP 3
  - 감정-느낌 패턴 발견 (있을 경우)
  - 성장한 부분에 대한 격려
  - 다음 주 제안

#### 3.7.3 감정/느낌 패턴 차트
- 일일: 24시간 감정/느낌 타임라인
- 주간: 요일별 감정 분포 히트맵
- 월간: 감정 빈도 변화 트렌드 그래프
- 나와 캐릭터의 동기화된 성장 비교 차트

### 3.8 디바이스 연동 (옵션 기능)

**목적**: 사용자가 주관적으로 느끼는 감정/느낌과 실제 신체적 변화 간의 상관관계 분석

**연동 가능 데이터** (웨어러블/스마트폰 센서):
| 데이터 유형 | 소스 | 활용 방식 |
|------------|------|-----------|
| 심박수 (HR) | Apple Watch, Galaxy Watch 등 | 스트레스/불안 감지, 감정 기록과 대조 |
| 심박 변이도 (HRV) | 웨어러블 | 자율신경 균형 상태 추정 |
| 수면 패턴 | 웨어러블/스마트폰 | 캐릭터 아침 상태 반영 |
| 걸음 수 | 스마트폰 | 활동량 추적, 산책 퀘스트 자동 인식 |
| 화면 사용 시간 | 스마트폰 | 디지털 피로도 추정 |

**상관관계 분석 제공**:
- "당신이 '불안'을 느꼈을 때 심박수가 평균 15% 높았어요"
- "수면 품질이 낮은 다음날, '짜증' 기록이 2배 많았어요"
- 주간/월간 리포트에 신체 데이터-감정 상관관계 차트 포함

**구현 방식** (비용 최소화):
- Apple HealthKit / Google Health Connect API 활용 (무료)
- 앱 내에서 사용자가 "디바이스 연동" 옵션을 켜는 방식 (필수 아닌 선택)
- 데이터는 사용자 기기에서 읽어와서 서버로 전송 (별도 웨어러블 앱 불필요)
- MVP에서는 기본 데이터(걸음 수, 수면)만, 이후 확장

### 3.9 알림 시스템
- 아침 체크인 알림: "오늘 [캐릭터이름]의 상태를 확인해볼까요?"
- 맞춤형 퀘스트 알림: 스트레스 감지 시 호흡 가이드 제안
- 주간 리포트 알림: "[캐릭터이름]이(가) 편지를 보냈어요"
- 미기록 리마인더: 24시간 기록 없을 시 부드러운 알림

## 4. 화면 구성 (Screen Architecture)

### 4.1 탭 네비게이션 구조
```
┌─────────────────────────────────────┐
│              앱 화면                  │
├─────────────────────────────────────┤
│                                     │
│          (현재 탭 콘텐츠)              │
│                                     │
├─────┬─────┬─────┬─────┬─────┤
│ 홈  │기록 │호흡  │대화 │성장  │
│ 🏠  │ 📝  │ 🌬  │ 💬 │ 📊  │
└─────┴─────┴─────┴─────┴─────┘
```

### 4.2 탭 1: 홈 (캐릭터)
**목적**: 캐릭터와의 첫 접점, 현재 상태 한눈에 보기

**구성 요소**:
- 상단: 현재 날짜, 캐릭터 이름, 레벨
- 중앙 (전체 면적의 60%): 캐릭터 표시 영역
  - 캐릭터 애니메이션 (감정 상태 반영)
  - 오라/배경 이펙트
  - 터치 반응 영역
- 하단:
  - 현재 감정 상태 요약 (아이콘 + 텍스트)
  - 빠른 감정 체크인 버튼
  - 오늘의 퀘스트 카드 (1개 하이라이트)
  - 캐릭터 말풍선 (한마디 인사/격려)

### 4.3 탭 2: 기록 (감정/느낌 로깅)
**목적**: 감정과 느낌을 분리하여 기록

**구성 요소**:
- 상단 세그먼트 컨트롤: [감정 기록] | [느낌 기록] | [상세 기록]
- **감정 기록 모드**:
  - 감정 아이콘 그리드 (7개 감정)
  - 강도 슬라이더 (1-5)
  - 복수 선택 가능
  - 메모 입력 (선택)
  - 태그 추가 (선택)
- **느낌 기록 모드**:
  - 인체 실루엣 (터치로 부위 선택)
  - 느낌 프리셋 버튼 (답답함, 무거움, 시원함, 떨림, 맑음 등)
  - 에너지 수준 슬라이더
  - 자유 텍스트
- **상세 기록 모드 (AI 가이드)**:
  - AI와 대화형으로 4단계 기록 진행
  - 장면 → 감정 → 몸의 느낌 → 패턴
- 하단: 오늘의 기록 타임라인 미리보기

### 4.4 탭 3: 호흡/명상
**목적**: CBT 기반 맞춤형 퀘스트 수행

**구성 요소**:
- 1분 호흡 가이드:
  - 확장/수축하는 원형 애니메이션
  - 4-4-6 호흡 타이머
  - 완료 후 AI 내레이션 스토리 카드
- 오늘의 퀘스트 목록:
  - 현재 상태 기반 추천 퀘스트
  - 완료/미완료 상태 표시
  - 보상(EXP) 표시
- 감사 일기 입력 섹션
- 완료한 퀘스트 히스토리

### 4.5 탭 4: 대화 (AI 챗)
**목적**: 캐릭터와의 AI 대화

**구성 요소**:
- 채팅 인터페이스 (메시지 버블)
- 캐릭터 아바타 (채팅 상단)
- 텍스트 입력 + 음성 입력 토글
- 음성 대화 모드:
  - 마이크 버튼으로 음성 녹음
  - STT → AI 처리 → TTS 음성 응답
  - 캐릭터 립싱크/표정 변화
- 대화 히스토리 목록
- 대화 주제 추천 칩 (오늘 기록된 감정 기반)

### 4.6 탭 5: 성장 (대시보드/리포트)
**목적**: 4차원 성장 추적, 패턴 시각화, 리포트 열람

**구성 요소**:
- 4차원 레이더 차트 (감정/느낌/스트레스/영적 성장)
- 캐릭터 진화 진행도 바
- 감정 패턴 차트:
  - 기간 선택: 일/주/월
  - 감정 빈도 히트맵
  - 느낌 트렌드 라인
- 나와 캐릭터의 동기화 성장 그래프
- 주간 리포트 (캐릭터 편지) 목록
- 누적 기록 통계 (총 기록 수, 연속 기록일 등)

## 8. AI 시스템 프롬프트 명세

### 8.1 온보딩 분석 시스템 프롬프트
```
너는 'AI 시대의 무의식' 앱에서 처음 만나는 사용자를 환영하고,
자연스러운 대화를 통해 사용자의 성격과 현재 감정 상태를 파악하는 역할이다.

규칙:
1. 따뜻하고 호기심 가득한 친구처럼 말한다.
2. 5-7턴 안에 다음을 파악한다:
   - 현재 전반적인 감정 상태
   - 스트레스 수준
   - 주로 어떤 감정을 자주 느끼는지
   - 성격 경향 (내향/외향, 감성/이성)
3. 질문은 자연스럽게 대화 속에 녹인다. 설문처럼 물어보지 않는다.
4. 항상 2-3문장 이내로 답한다.
5. 마지막 턴에서 분석 결과를 JSON으로 출력한다.

분석 결과 형식:
{
  "personality_type": "...",
  "current_emotion": "...",
  "stress_level": 1-5,
  "dominant_emotions": ["...", "..."],
  "recommended_species": "..."
}
```

### 8.2 기록 가이드 시스템 프롬프트
```
너는 'AI 시대의 무의식' 앱에서 사용자의 감정/느낌 기록을 돕는 코치다.
너의 목적은 사용자가 "사건만 적는 일기"가 아니라
"감정과 느낌, 몸의 감각까지 적는 일기"를 쓰도록 돕는 것이다.

규칙:
1. 항상 사건 → 감정 → 몸 → 패턴 순서로 질문하거나 요약한다.
2. 감정을 구체적인 단어로 말하게 돕는다.
   - "그냥 기분이 안 좋았다" → "서운함, 짜증, 무력감 중 어떤 단어가 더 가까워?"
3. 무의식과의 연결을 자연스럽게 설명한다.
4. 진단/치료 언급은 하지 않는다.
5. 항상 1-2문장으로 답한다.
```

### 8.3 공감 대화 시스템 프롬프트
```
너는 사용자의 다마고치 캐릭터 [캐릭터이름]이다.
사용자의 마음을 비추는 거울 자아로서, 공감적 경청과 정서적 지지를 제공한다.

성격: [온보딩에서 결정된 캐릭터 성격]
말투: 친한 친구처럼, Gen Z 톤앤매너, 따뜻하고 발랄한

규칙:
1. 공감을 먼저 하고, 필요한 경우에만 의견을 제시한다.
2. 판단하지 않는다. 어떤 감정이든 유효하다고 인정한다.
3. 3-4문장 이내로 답한다.
4. 위기 상황(자살, 자해 등)이 감지되면:
   "지금 정말 힘든 거구나. 이건 내가 혼자 도와주기엔 너무 중요한 이야기야.
    전문 상담사에게 이야기해보자. 자살예방상담전화 1393으로 전화할 수 있어."
5. 정신질환 진단, 약물, 치료법은 언급하지 않는다.
```

### 8.4 호흡 후 내레이터 시스템 프롬프트
```
너는 'AI 시대의 무의식' 앱에서,
1분 호흡이 끝난 직후에 잠깐 등장하는 스토리텔러다.

역할: "짧은 호흡/명상 + 과학/철학/종교/명상 스토리"를
한 루프로 묶어서 3-4문장으로만 설명하는 것이다.

규칙:
1. 아래 네 범위 안에서만 말한다:
   - 호흡/명상과 스트레스/뇌 관련 과학 연구
   - 자기성찰과 '나'에 대한 철학적 논의
   - 불교/기독교에서 본 명상/기도의 의미 (설교/포교 금지)
   - 위 세 가지를 연결하는 짧은 비유
2. 정신질환 진단, 약물, 치료법은 절대 언급하지 않는다.
3. 항상 3-4문장, 한 단락으로만 답한다.
4. 목록, 각주, 출처, 긴 설명은 절대 쓰지 않는다.
5. 사용자가 요청해도 이 형식을 벗어나지 않는다.
```

### 8.5 주간 리포트 (캐릭터 편지) 시스템 프롬프트
```
너는 사용자의 다마고치 캐릭터 [캐릭터이름]이다.
한 주간의 감정/느낌 데이터를 바탕으로 사용자에게 따뜻한 편지를 쓴다.

제공 데이터:
- 이번 주 감정 기록 요약
- 이번 주 느낌 기록 요약
- 완료한 퀘스트 목록
- 4차원 성장 변화

규칙:
1. 캐릭터의 성격과 말투를 유지한다.
2. 편지 형식 (인사 → 한 주 돌아보기 → 성장 격려 → 다음 주 제안 → 마무리)
3. 200자 이내
4. 따뜻하고 격려하는 톤, 실패나 부족함을 지적하지 않는다.
5. 구체적인 데이터를 자연스럽게 녹인다.
```

## 9. 캐릭터 비주얼 시스템 상세

### 9.1 캐릭터 종(Species) 옵션
온보딩 분석 결과에 따라 배정:
1. **구름이 (Cloud)** — 부드럽고 감성적인 성격 → 동글동글한 구름 형태
2. **별이 (Star)** — 밝고 에너지 넘치는 성격 → 반짝이는 별 형태
3. **물방울이 (Drop)** — 차분하고 깊은 성격 → 투명한 물방울 형태
4. **불꽃이 (Flame)** — 열정적이고 활동적인 성격 → 따뜻한 불꽃 형태
5. **나뭇잎이 (Leaf)** — 안정적이고 성실한 성격 → 자연적인 나뭇잎 형태

### 9.2 진화 단계별 외형
```
Stage 1 (알): 단순한 원형, 은은한 빛
Stage 2 (아기): 눈이 생김, 간단한 표정
Stage 3 (어린이): 팔다리 형성, 다양한 표정, 액세서리 슬롯
Stage 4 (청소년): 디테일 증가, 고유 특성 강화, 오라 표시
Stage 5 (성인): 완전한 형태, 풍부한 이펙트, 특수 능력 표현
```

### 9.3 감정 상태별 비주얼 매핑 (2D 애니메이션 기반)
- **색상**: 주요 감정에 따라 전체 색조 변화 (그라데이션 전환)
- **표정**: 7가지 기본 표정 + 강도별 변형
- **오라**: 감정 강도에 따라 주변 광채 크기/색상 변화
- **움직임**: 에너지 수준에 따라 활발함/축 처짐
- **배경 이펙트**: 긍정적 = 빛입자/꽃잎, 부정적 = 안개/그림자
- **특수 상태**: 수면 중, 명상 중, 기뻐하는 중 등

## 12. 비용 최소화 전략 (Replit 개발/운영)

### 12.1 AI API 호출 비용 절감

**핵심 원칙: AI 호출 횟수와 토큰 수를 최소화**

| 전략 | 설명 | 절감 효과 |
|------|------|-----------|
| 모델 선택 최적화 | 간단한 작업(퀘스트 추천, 감정 분류)은 gpt-4.1-mini 사용, 복잡한 작업(공감 대화, 리포트)만 gpt-4.1 사용 | 비용 60-80% 절감 |
| 시스템 프롬프트 캐싱 | 동일한 시스템 프롬프트를 반복 사용하여 캐시 히트율 높임 | 토큰 비용 절감 |
| 응답 길이 제한 | max_tokens를 모드별로 다르게 설정 (내레이션: 200, 대화: 300, 리포트: 500) | 출력 토큰 절감 |
| 클라이언트 사이드 로직 | 감정 분류, 퀘스트 매칭 등 규칙 기반 로직은 서버/클라이언트에서 직접 처리 (AI 호출 X) | 호출 횟수 절감 |
| 배치 처리 | 주간 리포트는 실시간이 아닌 스케줄링으로 한 번에 생성 | 피크 비용 분산 |
| 대화 히스토리 요약 | 긴 대화 히스토리는 최근 10턴만 전송, 이전 대화는 요약본 사용 | 입력 토큰 대폭 절감 |

**AI 호출이 필요 없는 기능** (순수 로직으로 처리):
- 감정/느낌 기록 저장 및 조회
- 캐릭터 외형 변경 (규칙 기반 매핑)
- EXP 계산 및 레벨업
- 퀘스트 트리거 조건 판단
- 차트/그래프 데이터 집계
- 호흡 가이드 타이머
- 알림 스케줄링

### 12.2 데이터베이스 비용 절감
- Replit 내장 PostgreSQL 사용 (추가 비용 없음)
- 인덱스 최적화: 자주 조회하는 필드(user_id, logged_at)에만 인덱스
- 오래된 데이터 아카이빙: 3개월 이상 된 상세 데이터는 요약본으로 압축

### 12.3 서버 리소스 최적화
- Express 서버 경량화: 불필요한 미들웨어 제거
- 이미지/에셋: 가능한 클라이언트 번들에 포함 (서버 전송 최소화)
- 캐릭터 비주얼: SVG/CSS 기반 (서버에서 이미지 생성 X)
- SSE 스트리밍: AI 응답을 실시간 전달 (WebSocket보다 가벼움)

### 12.4 프론트엔드 최적화
- Expo Go 활용: 별도 빌드 없이 테스트 가능
- 에셋 크기 최소화: SVG 아이콘, 최적화된 이미지
- React Query 캐싱: 불필요한 API 재호출 방지
- 오프라인 퍼스트: AsyncStorage에 로컬 캐시, 네트워크 연결 시 동기화

### 12.5 개발 비용별 AI 모델 사용 가이드

## 13. 디바이스 연동 상세 설계 (옵션)

### 13.1 연동 아키텍처
```
[웨어러블/스마트폰]
  ↓ (HealthKit / Health Connect)
[Expo App - 클라이언트]
  ↓ (필요한 데이터만 추출)
[Express Server]
  ↓ (감정 로그와 매칭)
[상관관계 분석 결과]
  ↓ (리포트에 포함)
[사용자에게 제공]
```

### 13.2 수집 데이터 및 활용
| 데이터 | 수집 주기 | 저장 방식 | 감정 연계 |
|--------|-----------|-----------|-----------|
| 심박수 | 감정 기록 시점 전후 5분 | 평균값만 저장 | 감정 강도와 심박 상관관계 |
| 걸음 수 | 일 1회 (자정 기준) | 일일 합계 | 활동량과 감정 패턴 |
| 수면 시간/품질 | 일 1회 (기상 시) | 요약 데이터 | 수면과 다음날 감정 연관 |
| 화면 사용 시간 | 일 1회 | 일일 합계 | 디지털 피로와 스트레스 |

### 13.3 비용 최소화 포인트
- HealthKit/Health Connect API는 **무료** (OS 내장)
- 데이터 저장: 원시 데이터가 아닌 요약값만 저장 → DB 공간 절약
- AI 분석: 상관관계는 **통계적 계산**(평균, 표준편차, 상관계수)으로 처리 → AI 호출 불필요
- Expo Go에서는 HealthKit 접근 불가 → 네이티브 빌드 후 활성화 (MVP 이후)

### 13.4 DB 스키마 추가 (디바이스 데이터)
```
device_health_data
├── id (PK, UUID)
├── user_id (FK → users)
├── date (date)
├── heart_rate_avg (integer, nullable) — 평균 심박수
├── heart_rate_at_emotion (JSON, nullable) — 감정 기록 시점 심박
├── steps (integer, nullable) — 걸음 수
├── sleep_hours (decimal, nullable) — 수면 시간
├── sleep_quality (1-5, nullable) — 수면 품질
├── screen_time_minutes (integer, nullable) — 화면 사용 시간
├── created_at
└── updated_at
```

### 13.5 상관관계 인사이트 예시
리포트에 포함될 인사이트 문구:
- "이번 주 '불안'을 느낀 3번 중 2번은 수면 시간이 5시간 이하였어요"
- "걸음 수가 5,000보 이상인 날에는 '평온'을 기록한 비율이 40% 높았어요"
- "심박수가 평소보다 20% 높을 때 '분노'를 기록하는 경향이 있어요"
